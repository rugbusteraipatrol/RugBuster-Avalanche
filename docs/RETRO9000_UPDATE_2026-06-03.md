# Retro9000 Update - 2026-06-03

RugBuster Apex has moved from a public Avalanche scanner demo into a live
security tooling stack for Avalanche builders.

## What Changed

- Live Avalanche builder API is online:
  `https://web-production-376bf.up.railway.app`
- Public cache score endpoint:
  `GET /score?address=0x...`
- Protected deep scan endpoint:
  `GET /scan?address=0x...` with `X-API-Key`
- Current AVAX classifier:
  `weighted_v2`
- Private AVAX scan corpus is stored in Postgres and continues to grow.
- Verified Avalanche mainnet registry remains live:
  `0x5F30276B3A5079E088Ec3072884286de5a868355`
- Website now documents the builder API layer and links API documentation.

## Why It Matters For Avalanche

RugBuster Apex is infrastructure tooling, not only a scanner UI. The stack now
has three layers:

1. Collector: continuously builds private Avalanche token evidence.
2. API: gives builders compact risk labels without exposing the raw corpus.
3. Registry: records reviewer-published token safety attestations on-chain.

This gives wallets, launchpads, DEX interfaces, dashboards, and future
Avalanche L1 security products a path to query token risk intelligence before
users interact with dangerous contracts.

## Next Milestone

The next milestone is RugBuster Sentinel: proactive Avalanche monitoring that
turns the existing collector, API, alerts, and registry into a continuously
running threat-intelligence layer for builders.

Sentinel will start as monitoring infrastructure and later evolve toward an
Avalanche L1 security layer.

For the Avalanche L1s & Infrastructure Tooling round, the project should be
submitted as developer security tooling for Avalanche L1 builders. RugBuster
Apex does not currently claim a production L1 deployment; the shipped product is
scanner, registry, API, alerting, and evidence infrastructure with planned
ICM/ICTT-aware integrations.
